# Dev Kumpulan Skill

Kumpulan skill untuk development harian — grill, spec, breakdown, implement, review, debug, migrasi. **Full local**: semua artifact di `.agent-docs/`, tidak publish ke GitHub/tracker eksternal.

## Install

Butuh pi agent. Install global ke `~/.pi/agent/skills/` (semua project) — skill ini dirancang all-or-nothing karena saling depend via `setup-workflow` marker.

```bash
curl -L https://raw.githubusercontent.com/Divarizky/dev-workflow-skills/main/install.sh | bash
```

Flags:
- `./install.sh --force` — overwrite semua skill tanpa prompt
- `./install.sh --help` — usage

Setelah install, **restart pi agent** (atau reload skills) supaya skill baru ke-pickup.

Verifikasi:

```bash
ls ~/.pi/agent/skills/
# harus muncul: ask-me/  bug-diagnosis/  code-review/  handoff/  implement/
#                improve-architecture/  project-migration/  prototype/
#                setup-workflow/  status/  to-issues/  to-prd/
```

## Quick Start

1. **Baru pertama kali di repo** → `setup-workflow`
2. **Bingung pilih skill mana / mulai dari mana** → `ask-me` (auto-trigger, atau invoke manual)
3. **Liat progress kerja** → `status`

Fungsi/trigger detail tiap skill: lihat `description` di masing-masing `SKILL.md` — satu sumber kebenaran, tidak diduplikasi di sini.

## Flow

```
setup-workflow (sekali per repo)
       │
       ▼
   ask-me (jalur utama + grill — auto-trigger)
       │
       ├──→ implement ──chain──→ code-review
       │      (fitur kecil, hasil grill langsung)
       │
       └──→ [grill dalam] → to-prd ──chain──→ to-issues ──chain──→ implement
                              (fitur besar / ambigu)

Jalur lain:
  bug-diagnosis          — debug 6-phase (auto-trigger)
  improve-architecture   — health check modul
  project-migration      — migrasi project lama → baru
  prototype              — throwaway code untuk validasi desain (before to-issues)
  handoff                — crossing sessions
  status                 — snapshot posisi kerja
```

**Referensi lintas skill (manual chain)** — alih-alih auto-invoke, tiap skill membaca SKILL.md skill berikutnya dan menjalankan step-nya langsung di sesi yang sama:
- `to-prd` → setelah PRD di-approve user, baca `to-issues/SKILL.md` dan jalankan Step 2-4
- `to-issues` → setelah tasks.md siap, baca `implement/SKILL.md` dan jalankan Step 1-5
- `implement` → setelah implementasi, baca `code-review/SKILL.md` dan jalankan Step 1-5

Aturan sesi (single vs multi-session) ada di `ask-me` — section Aturan Sesi.

## Artifact Schema (`.agent-docs/`)

**Single source of truth** untuk struktur & field tiap artifact yang disimpan lokal di repo.

### `project-meta.md` — Setup marker

Dibuat oleh `setup-workflow`. Keberadaannya = penanda setup sudah jalan. Skill lain gate-check file ini.

```yaml
---
status: <new|existing>
setup_date: <YYYY-MM-DD>
```

### `CONTEXT.md` — Domain glossary

Vocabulary & terminology project. Build via `ask-me` grill. Field:
- **Terms** — istilah domain + definisi
- **Aliases to avoid** — term ambigu yang perlu dihindari
- **Concepts** — conceptual model project

### `ADR.md` — Architecture Decision Records

Keputusan yang lolos 3 filter ADR (hard to reverse, surprising without context, real trade-off — lihat `VOCABULARY.md`). Kosong untuk existing project sampai keputusan besar muncul. Update by `ask-me` grill dalam.

Format per-entry:
```
## [Date] — [Decision Title]

Context: ...
Decision: ...
Consequences: ...
```

### `.scratch/<feature-slug>/prototype-decision.md` — Prototype answer

Generated by `prototype` atau `implement` (kalo task prototype). Capture hasil validasi desain.

```markdown
# Prototype Decision — <nama>

**Question:** <satu kalimat>
**Branch:** LOGIC | UI
**Answer:** <kesimpulan>
**Date:** <YYYY-MM-DD>
**Prototype branch:** <nama branch>

**What was validated:** <bagian yang diambil ke real code>
**What was discarded:** <bagian yang di-throwaway>
```

### `TDD.md` — Test-first discipline

Referensi copy dari `setup-workflow/seeds/TDD.md`. Isi tetap, tidak project-specific. RED-GREEN-REFACTOR: satu test → minimal impl → refactor.

### `issue-tracker.md` — Feature index

Maintained by `to-issues`, `implement`. Track per-feature status + progress.

```yaml
tracker: local
features:
  - slug: <feature-slug>
    status: <open|done>
    source: <to-prd|ask-me|manual>
    created: <YYYY-MM-DD>
    updated: <YYYY-MM-DD>
    task_count: <total task>
    task_done: <task selesai>
```

### `.scratch/<feature-slug>/PRD.md` — Feature spec (full)

Generated by `to-prd`. Sections:

```yaml
---
version: <semver>
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
source: to-prd
status: <draft|approved|superseded>
---
```

- **Problem** — problem statement
- **Solution** — proposed solution
- **User Stories** — dengan label prioritas (MUST/SHOULD/NICE)
- **Acceptance Criteria** — kondisi konkret fitur selesai
- **Implementation Decisions** — dipisah: Final (butuh ADR) vs Open (perlu validasi)
- **Testing Decisions** — Seam(s), Test Strategy, Environment
- **Out of Scope** — explicitly exclude
- **Further Notes** — follow-up, open questions

### `.scratch/<feature-slug>/tasks.md` — Task breakdown

Generated by `to-issues` atau `implement` (tracking minimal). Tracks task progress.

Group labels (bold text, bukan heading markdown):
- **Queue** — task belum dikerjakan, urut priority (critical→low) + unblocked duluan
- **In Progress** — task sedang dikerjakan
- **Done** — task selesai
- **Superseded** — task diganti (split/merge), tidak pernah selesai

Checkbox status: `[ ]` = not started / in progress, `[x]` = done

Task format:
```
- [ ] TASK-N | <name> | Depends: <none|TASK-M> | Priority: critical|high|medium|low
    Detail: <end-to-end behavior>
    Done:
    - [ ] <kriteria 1>
```

Each task = vertical slice (tembus semua layer, tidak horizontal per-layer).

### `handoffs/<timestamp>-<slug>.md` — Handoff doc

Generated by `handoff`. Compact sesi kerja buat lanjutan di session baru.

**Rules:**
- Reference-only (link by path, jangan duplikasi konten besar)
- Snippet kecil OK (<5 baris), konten besar → link saja
- Sections: Context, Progress, Blockers, Suggested Skills (wajib)

### `.scratch/migration/meta.md` — Migration state (transient)

Maintained by `project-migration`. Fields:
- `migration_source` — project lama
- `migration_source_type` — GitHub / GitLab / local / dll
- `migration_approach` — phased / big-bang / hybrid
- `cutover_strategy` — timing & validation

**Transient** — dihapus setelah migrasi selesai & validasi lolos. Terpisah dari `project-meta.md` (marker permanen).

## Tips

- **User sudah sebut nama skill explicit** → panggil skill itu langsung, `ask-me` bisa dilewati.
- **Setup hanya sekali per repo**, invoke ulang cuma kalau reset.
- **Handoff untuk ganti sesi**, compact untuk lanjut thread yang sama.
